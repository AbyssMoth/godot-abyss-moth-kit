# Abyss Moth Kit

Umbrella-инструмент студии Abyss Moth для Godot 4.7. Один аддон, который ставит другие студийные аддоны из GitHub, трекает их версии и (в следующих фазах) инициализирует структуру проекта и базовые настройки.

Лежит в `addons/abyss_moth/abyss_moth_kit/`. Сами устанавливаемые аддоны (reactive, save и будущие) остаются отдельными репозиториями и ставятся сиблингами в `addons/abyss_moth/<name>/`.

Репозиторий студийных аддонов под движок именуется с префиксом `godot-` (`godot-reactive`, `godot-save`), чтобы в будущем можно было фильтровать репо по движку. Сам kit: `godot-abyss-moth-kit`.

## Установка для художников (без кода, без терминала)

Поставить нужно только ЭТОТ один аддон. Дальше все остальные ставятся кнопками из его панели, руками ничего создавать не надо.

1. Открыть релизы: https://github.com/AbyssMoth/godot-abyss-moth-kit/releases
2. В последнем релизе, в разделе Assets, скачать `abyss_moth_kit.zip` (именно его, НЕ "Source code").
3. В Godot вверху открыть вкладку "Библиотека ассетов" (AssetLib, рядом с 2D / 3D / Скрипт).
4. Нажать "Импорт..." (Import), выбрать скачанный `abyss_moth_kit.zip`, нажать "Установить".
5. Настройки проекта -> Плагины -> включить `Abyss Moth Kit`.
6. Справа появится вкладка "Abyss Moth". Остальные аддоны ставятся кнопками оттуда, папку `addons/abyss_moth/` создавать вручную не нужно.

`abyss_moth_kit.zip` из релиза имеет правильную структуру `addons/abyss_moth/abyss_moth_kit/`, поэтому импортируется куда надо. Обычный "Source code (zip)" с GitHub так не сработает: у него файлы в корне и Godot не увидит их как аддон (папка должна называться `abyss_moth_kit` и лежать именно в `addons/`).

## Bootstrap для программистов (терминал)

Сам kit нельзя поставить через собственный dock в пустом проекте, поэтому первый раз он заводится вручную одной командой из корня проекта:

```bash
git clone --depth 1 https://github.com/AbyssMoth/godot-abyss-moth-kit \
  addons/abyss_moth/abyss_moth_kit && rm -rf addons/abyss_moth/abyss_moth_kit/.git
```

`rm -rf .git` обязателен, иначе вложенный `.git` запутает git родительского проекта.

Дальше включите плагин в Project Settings -> Plugins (`Abyss Moth Kit`). Появится dock-панель справа.

## Что умеет

- Прокручиваемый список студийных аддонов из `data/catalog.json` со статусом и цветом (зелёный - актуально, жёлтый - есть обновление, серый - нет/нет сети, красный - ошибка).
- Установка на каждой строке: качает zip ветки с GitHub (без зависимости от системного git), распаковывает в `addons/abyss_moth/<name>/`, включает плагин.
- `Установить набор base`: ставит пресет, пропуская уже установленные (защита от повторной установки).
- `Проверить обновления`: сравнивает локальный commit sha с upstream (через `git ls-remote`, фолбэк REST) и пишет понятный итог в лог.
- `Обновить все`: переустанавливает аддоны, у которых найдено обновление.
- Карточка аддона по клику на имя: описание, ветка, установленная версия и commit, кнопка `Открыть на GitHub` (можно вспомнить инфу не выходя в браузер).
- `data/abyss_lock.json` пишется при установке (owner/repo/branch/installed_plugin_version/installed_sha/installed_at) и коммитится как источник истины для трекинга версий.

## Версии и обновления

Трекинг идёт по **commit sha ветки**, а не по version из plugin.cfg (студийные аддоны могут долго стоять на 0.1.0 при разных коммитах). `installed_plugin_version` сохраняется и показывается для удобства, но решение об обновлении принимается по sha. Поэтому даже если вы только подняли version и закоммитили - это меняет sha ветки, и `Проверить обновления` это увидит.

Если `installed_sha` пустой (например аддон ставился, когда упёрлись в лимит REST 60/час), переустановите аддон 1 раз - sha зафиксируется через `git ls-remote`, и дальше трекинг работает.

## Логи

История действий пишется в `logs/kit.log` (рядом с аддоном). Папка `logs/` под локальным `.gitignore`, в репозиторий не попадает. Если что-то пошло не так - откройте этот файл.

## Структура

```text
abyss_moth_kit/
  plugin.cfg / plugin.gd      # @tool EditorPlugin: только жизненный цикл dock
  core/
    github_client.gd          # HTTPRequest: загрузка zip ветки
    version_check.gd          # sha ветки: git ls-remote, фолбэк REST
    installer.gd              # download -> распаковка -> swap -> lock
    logger.gd                 # файловый лог logs/kit.log
  data/
    catalog.json              # реестр аддонов + пресеты (редактируемый)
    abyss_lock.json           # что установлено (генерируется, коммитится)
  logs/                       # история (gitignored)
  ui/
    dock_panel.gd             # панель: список, установка, обновления, карточка, лог
```
